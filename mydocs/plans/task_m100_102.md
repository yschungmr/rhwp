## 현상

`hwpspec.hwp` 18페이지(page=5) 마지막에 s2:pi=65 첫 줄이 편집용지 높이를 초과해 클리핑됨.

```
LAYOUT_OVERFLOW: page=5, col=0, para=65, type=PartialParagraph, y=1047.3, bottom=1034.1, overflow=13.2px
```

- 18페이지: `PartialParagraph pi=65 lines=0..1` — 첫 줄이 페이지 바닥(1034.1px)을 13.2px 초과
- 19페이지: `PartialParagraph pi=65 lines=1..2` — 2번째 줄 연속

## 근본 접근

높이 계산 오차는 개별 케이스마다 수정으로 줄여왔으나 완벽할 수 없다.
한컴과 같이 **배치 확정 시점에 줄 단위로 페이지 경계 초과 여부를 검사하고, 초과 시 해당 줄부터 다음 페이지로 이동**하는 방어 로직을 추가한다.

- 1차 방어: `current_height` 정확도 향상 (기존)
- 2차 방어: 줄 배치 시점에 실제 y + 줄높이 > page_bottom이면 페이지 이동 (신규)

## 적용 범위

- **대상**: 일반 문단의 LINE_SEG(줄) 단위 배치
- **제외**: 표, 글상자 등 (별도 분할 로직 존재)

## 구현 방침

방어 로직 코드에 다음 취지의 주석을 명시한다:

```rust
// TODO: 높이 계산 오차에 대한 임시 방어 로직.
// current_height 누적이 정확해지면 이 코드는 제거 가능하다.
```

근본적인 높이 계산 수정이 완료된 시점에 제거를 검토한다.

## 수행 범위

1. 페이지네이션 엔진에서 문단 줄 배치 처리 코드 파악
2. 줄 단위 방어 로직 설계 및 구현계획서 작성
3. 구현 및 검증

## 검증 기준

- `export-svg samples/hwpspec.hwp -p 17 --debug-overlay`: pi=65 클리핑 없음
- LAYOUT_OVERFLOW page=5 para=65 제거
- 전체 226개 샘플 새 회귀 0건
- 단위 테스트 전체 통과

## 제약 조건

- 표·글상자 배치 로직 변경 없음
- 전체 샘플 새 회귀 0건 유지
- 수정 범위 최소화

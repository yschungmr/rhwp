# Task #85: iOS Safari 뷰어 — 수행계획서 (v3)

## 목표

iOS에서 HWP 문서를 열 수 있는 네이티브 앱을 개발한다.

## 배경

iOS Safari Web Extension의 구조적 제약으로 확장 내 뷰어가 동작하지 않음:
- `tabs.create()` + 확장 페이지: JS 미실행 (silent failure)
- iframe + `safari-web-extension://` URL: 네트워크 차단
- Safari 확장 설정 UX가 매우 불편 (일반 사용자 사용 불가 수준)

## 유지 사항

- **macOS Safari 확장**: 정상 동작, 그대로 유지 (`rhwp-safari/`)
- **Chrome/Edge 확장**: 심사 중, 그대로 유지 (`rhwp-chrome/`)

## 해결 방향: iOS 전용 독립 네이티브 앱

**별도 Xcode 프로젝트**로 iOS HWP 뷰어 앱을 개발한다.
- `rhwp-safari/`는 macOS Safari 확장 전용으로 유지 (수정 없음)
- `rhwp-ios/`에 새 프로젝트 생성
- WKWebView + WKURLSchemeHandler로 WASM 뷰어 로드
  - `file://`에서 `type="module"` CORS 차단 → 커스텀 스킴으로 해결
- iOS Safari에서 WASM 동작은 확인됨

### 핵심 기능

1. **WKWebView + WKURLSchemeHandler + WASM 뷰어**: 커스텀 스킴으로 로컬 파일 서빙
2. **Share Sheet 연동**: Safari 등에서 "rhwp로 열기" 지원
3. **.hwp/.hwpx 파일 연결**: 파일 앱에서 탭하면 rhwp로 열기
4. **UIDocumentPickerViewController**: 앱 내 파일 선택

## 구현 단계

### 1단계: Xcode 프로젝트 생성 + WKWebView 뷰어 동작

- `rhwp-ios/` 에 iOS 전용 Xcode 프로젝트 생성 (SwiftUI 또는 UIKit)
- WKURLSchemeHandler로 커스텀 스킴 구현 (CORS/module 우회)
- rhwp-studio 뷰어 리소스를 앱 번들에 포함
- WASM 초기화 + 파일 선택 → HWP 렌더링 확인

### 2단계: Share Sheet + 파일 연결

- UTType 등록, CFBundleDocumentTypes
- Share Extension
- SceneDelegate에서 외부 파일 처리

### 3단계: UX + 검증 + 배포

- 앱 아이콘, 시작 화면
- iOS 실제 기기 테스트
- App Store 배포 준비

## 영향 범위

- `rhwp-ios/` — 신규 프로젝트 (독립)
- `rhwp-safari/` — 변경 없음 (macOS 확장 유지)
- 뷰어 리소스: `rhwp-safari/dist/` 또는 별도 빌드에서 복사

## 참고

- iOS Safari에서 웹버전 rhwp-studio WASM 동작 확인됨
- 기존 Xcode 프로젝트에 iOS 앱 타겟이 이미 존재

## 현상

`.github/workflows/deploy-pages.yml`에 `tags: ['v*']` 트리거가 설정되어 있어,
릴리즈 태그 push 시 GitHub Pages 배포가 중복 실행된다.

GitHub Pages 환경 보호 규칙이 태그 기반 배포를 차단하여 워크플로우가 실패로 표시됨:

```
X Tag "v0.7.0" is not allowed to deploy to github-pages due to environment protection rules.
```

`main` branch push 시 이미 정상 배포가 실행되므로 태그 트리거는 불필요하다.

## 수정 내용

`.github/workflows/deploy-pages.yml`의 트리거에서 `tags: ['v*']` 제거:

```yaml
# 수정 전
on:
  push:
    branches: [main]
    tags: ['v*']
  workflow_dispatch:

# 수정 후
on:
  push:
    branches: [main]
  workflow_dispatch:
```

## 수행 범위

1. `local/task105` 브랜치 생성
2. `deploy-pages.yml` 수정
3. main에 직접 커밋 (단순 1줄 수정)

## 검증 기준

- 다음 릴리즈 태그 push 시 `deploy-pages.yml`이 실행되지 않음
- `main` push 시 `deploy-pages.yml` 정상 실행 확인

## 제약 조건

- 워크플로우 로직 변경 없음, 트리거만 수정

# Task #88: macOS Safari 확장 HWP 썸네일 기능 — 수행계획서

## 목표

Chrome 확장(Task #86)의 HWP 썸네일 자동 추출 기능을 macOS Safari 확장에 포팅한다.

## Chrome 확장 구현 분석

### 구조
- `sw/thumbnail-extractor.js`: HWP(CFB)/HWPX(ZIP) 파일에서 PrvImage 추출 (순수 JS, WASM 불필요)
- `sw/message-router.js`: `extract-thumbnail` 메시지 핸들러
- `content-script.js`: 호버 카드에 썸네일 표시 + background에 추출 요청

### 기술
- HWP(CFB): FAT 체인 파싱 → PrvImage 스트림 추출
- HWPX(ZIP): Central Directory → Preview/PrvImage.* → DecompressionStream(deflate)
- 캐시: URL 기반 LRU 100건
- 이미지: PNG/BMP/GIF 감지 → base64 dataUri

### Safari 호환성
- DecompressionStream: Safari 16.4+ 지원 ✅
- 순수 JS 구현이므로 API 차이 없음

## 구현 단계

### 1단계: thumbnail-extractor.js 포팅

- Chrome의 `thumbnail-extractor.js`를 Safari background.js에 통합 (ES module 아닌 인라인)
- `extract-thumbnail` 메시지 핸들러 추가
- sender 검증 (content script만 허용)

### 2단계: content-script.js 썸네일 연동

- 호버 카드에 썸네일 표시 (Chrome과 동일)
- `extract-thumbnail` 메시지로 background에 추출 요청
- URL 기반 캐시

### 3단계: 검증

- macOS Safari에서 HWP/HWPX 파일 호버 시 썸네일 표시 확인
- 캐시 동작 확인
- macOS Safari 기존 기능 회귀 테스트
